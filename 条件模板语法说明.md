# 数据库条件模板语法说明

## 概述

数据库插件提供了条件模板功能，允许根据条件动态显示或隐藏提示词内容。该功能通过 `<if>` 标签实现，支持关键词匹配和表格数值比较两种条件类型。此外，还提供随机数生成功能。

## 功能开关

在"数据库更新"设置区域，勾选"启用条件模板功能"复选框即可开启此功能。

## 基本语法

### 1. 关键词匹配 (`seed`)

根据数据库表格内容（除纪要表外）和 `$6`（上轮剧情规划数据）中是否包含指定关键词来决定是否显示内容。

```
<if seed="关键词表达式">条件内容</if>
```

### 2. 表格数值比较 (`cell`)

根据数据库表格中的数值来决定是否显示内容。

```
<if cell="表格名/行名/列名 比较运算符 数值">条件内容</if>
```

### 3. 随机数生成 (`random`)

在提示词中生成指定范围内的随机整数。

```
<random min="最小值" max="最大值" />
```

**示例**：
```
<!-- 生成 1-100 之间的随机数 -->
🎲 本轮随机数：<random min="1" max="100" />

<!-- 生成 1-6 之间的随机数（模拟骰子） -->
🎲 骰子结果：<random min="1" max="6" />
```

## 关键词匹配详解

### 基本用法

```
<if seed="战斗">这里的内容只有在消息中包含"战斗"时才会显示</if>
```

### 逻辑运算符

| 运算符 | 含义 | 示例 |
|--------|------|------|
| `&` | 与（AND） | `战斗&魔法` - 同时包含两个关键词 |
| `,` | 或（OR） | `战斗,魔法` - 包含任意一个关键词 |
| `!` | 非（NOT） | `!战斗` - 不包含该关键词 |

### 组合示例

```
<!-- 同时包含"战斗"和"魔法" -->
<if seed="战斗&魔法">
  你正在施展战斗魔法！
</if>

<!-- 包含"战斗"或"逃跑" -->
<if seed="战斗,逃跑">
  场景变得紧张起来。
</if>

<!-- 不包含"安全" -->
<if seed="!安全">
  危险潜伏在四周。
</if>

<!-- 复杂组合：包含("战斗"且"魔法") 或 ("逃跑"且"追击") -->
<if seed="战斗&魔法,逃跑&追击">
  这是一场追逐战！
</if>
```

### 模糊匹配

关键词匹配支持模糊匹配，会自动匹配包含该词的更长词语：

- 关键词 `战斗` 可以匹配 `战斗中`、`战斗开始`、`激烈战斗` 等
- 匹配不区分大小写

## 表格数值比较详解

### 基本语法

```
<if cell="表格名/行名/列名 比较运算符 数值">条件内容</if>
```

### 比较运算符

| 运算符 | 含义 | 示例 |
|--------|------|------|
| `>` | 大于 | `> 50` |
| `<` | 小于 | `< 100` |
| `>=` | 大于等于 | `>= 30` |
| `<=` | 小于等于 | `<= 70` |
| `==` | 等于 | `== 100` |
| `!=` | 不等于 | `!= 0` |

### 示例

```
<!-- 魔力值大于30时显示 -->
<if cell="状态表/主角/魔力值 > 30">
  你有足够的魔力施放高级魔法。
</if>

<!-- 好感度小于等于30时显示 -->
<if cell="关系表/陈默/好感度 <= 30">
  陈默对你保持警惕。
</if>

<!-- 等级等于10时显示 -->
<if cell="属性表/主角/等级 == 10">
  恭喜你达到了10级！
</if>

<!-- 字符串相等比较（支持中文） -->
<if cell="全局数据表/本轮故事模式 == 战斗模式">
  当前处于战斗状态！
</if>

<!-- 字符串不等比较 -->
<if cell="全局数据表/本轮故事模式 != 战斗模式">
  当前不是战斗状态。
</if>
```

### 行列颠倒自动匹配

如果指定的"行名/列名"找不到，系统会自动尝试颠倒查找：

```
<!-- 以下两种写法等效 -->
<if cell="状态表/主角/魔力值 > 30">...</if>
<if cell="状态表/魔力值/主角 > 30">...</if>
```

### 单行表格支持

对于只有一行数据的表格（如「全局数据表」），系统会自动识别并支持直接用列名获取值：

```
<!-- 全局数据表只有一行数据，可以直接用「表格名/列名」格式 -->
<if cell="全局数据表/本轮故事模式 == 战斗模式">
  当前处于战斗状态！
</if>

<!-- 也可以用「表格名/任意标识/列名」格式，系统会自动找到对应的列 -->
<if cell="全局数据表/状态/本轮故事模式 == 战斗模式">
  当前处于战斗状态！
</if>
```

### 字符串比较说明

- 字符串比较支持 `==`（相等）和 `!=`（不等）运算符
- 字符串比较是**精确匹配**，区分大小写
- 如果单元格值是数字类型，比较值也是数字，则进行数值比较
- 否则进行字符串比较

## else 分支

当条件不满足时，可以显示备选内容：

```
<if seed="战斗">
  战斗场景内容
<else>
  非战斗场景内容
</if>
```

```
<if cell="状态表/主角/魔力值 > 30">
  魔力充足，可以施放高级魔法。
<else>
  魔力不足，只能使用普通攻击。
</if>
```

## 嵌套条件

条件可以嵌套使用，实现更复杂的逻辑：

```
<if seed="战斗">
  <if cell="状态表/主角/魔力值 > 30">
    你施放了一个强大的魔法攻击！
  <else>
    魔力不足，你只能使用普通攻击。
  </if>
<else>
  <if seed="休息">
    你找了一个安静的地方休息。
  <else>
    你继续探索这个区域。
  </if>
</if>
```

**注意**：嵌套深度有限制，建议不超过 5 层。

## 完整示例

### 示例1：根据场景类型显示不同内容

```
# 场景描述

<if seed="战斗">
  ## 战斗场景
  当前处于战斗状态，请描述你的战斗行动。
  <if cell="状态表/主角/生命值 <= 30">
    ⚠️ 警告：你的生命值较低！
  </if>
<else>
  <if seed="商店">
    ## 商店场景
    你可以购买或出售物品。
  <else>
    ## 探索场景
    你正在探索这个区域。
  </if>
</if>
```

### 示例2：根据角色状态显示人设

```
# 角色状态

<if cell="属性表/陈默/堕落值 <= 10">
  ## 阶段1：迷茫的旁观者
  陈默保持着纯真的心态，对世界充满好奇。
<else>
  <if cell="属性表/陈默/堕落值 <= 30">
    ## 阶段2：初涉黑暗
    陈默开始感受到内心的变化。
  <else>
    <if cell="属性表/陈默/堕落值 <= 60">
      ## 阶段3：黑暗觉醒
      陈默的内心逐渐被黑暗侵蚀。
    <else>
      ## 阶段4：堕落完成
      陈默已经完全堕入黑暗。
    </if>
  </if>
</if>
```

### 示例3：复杂条件组合

```
# 剧情分支

<if seed="战斗&魔法">
  <if cell="状态表/主角/魔力值 > 50">
    你施放了一个强大的范围魔法，敌人受到重创！
  <else>
    你尝试施放魔法，但魔力不足，效果大打折扣。
  </if>
<else>
  <if seed="战斗,逃跑">
    <if cell="状态表/主角/速度 > 敌人/速度">
      你成功逃脱了战斗！
    <else>
      敌人追上了你，战斗无法避免。
    </if>
  <else>
    平静的时光流逝...
  </if>
</if>
```

## 与 st-prompt-template 的关系

数据库插件的条件模板功能与酒馆的 st-prompt-template 插件协同工作：

1. **处理顺序**：提示词先经过 st-prompt-template 处理（EJS 语法），再由数据库的条件模板处理（`<if>` 语法）
2. **功能互补**：
   - st-prompt-template：处理 `<% %>` EJS 语法，可访问酒馆变量系统
   - 数据库条件模板：处理 `<if>` 语法，可访问数据库表格数据

### 示例：两者配合使用

```
<%# 先由 st-prompt-template 处理 EJS 语法 %>
<% if (getvar("some_var") === "value") { %>
  这里的内容会先被 EJS 处理
<% } %>

<%# 再由数据库条件模板处理 %>
<if seed="战斗">
  这里的内容会被条件模板处理
</if>
```

## 注意事项

1. **标签闭合**：所有 `<if>` 标签必须正确闭合 `</if>`
2. **大小写敏感**：关键词匹配不区分大小写，表格名/行名/列名区分大小写
3. **数值类型**：表格数值比较时，非数值内容会被视为 0
4. **性能考虑**：复杂的嵌套条件可能影响处理速度，建议控制嵌套深度
5. **错误处理**：语法错误时，整个 `<if>` 块会被保留原样输出

## 调试技巧

1. 检查控制台日志，查看条件匹配结果
2. 使用简单的测试条件验证功能是否正常
3. 逐步添加复杂条件，确保每一步都正确工作

## 随机数生成详解

### 基本语法

```
<random min="最小值" max="最大值" />
```

### 功能说明

- 在提示词处理时生成一个指定范围内的随机整数
- 生成的随机数会替换整个 `<random>` 标签
- 随机数在每次剧情推进时重新生成

### 使用场景

#### 1. 随机事件判定

```
## 随机事件判定
🎲 **本轮随机数**：<random min="1" max="100" />

**事件判定表**：
- 1-10：⚠️ 重大意外事件（负面）
- 11-30：📊 小意外/波折
- 31-70：✨ 平稳推进
- 71-90：🍀 小机遇/惊喜
- 91-100：🌟 重大机遇事件（正面）
```

#### 2. 骰子模拟

```
<!-- D20 骰子 -->
🎲 D20: <random min="1" max="20" />

<!-- D6 骰子（3个） -->
🎲 3D6: <random min="1" max="6" /> + <random min="1" max="6" /> + <random min="1" max="6" />
```

#### 3. 随机选择

```
<!-- 从 1-4 中随机选择一个选项 -->
随机选项编号：<random min="1" max="4" />
```

### 与条件模板配合使用

随机数生成在条件模板解析**之前**执行，因此可以在条件中使用生成的随机数：

```
🎲 **随机数**：<random min="1" max="100" />

<if seed="重大意外">
  触发了负面事件！
<else>
  <if seed="小意外">
    遇到了一些小麻烦...
  <else>
    一切顺利。
  </if>
</if>
```

**注意**：由于随机数是数值而非关键词，如果需要在条件判断中使用，建议在提示词中直接根据数值范围描述结果，而不是用 seed 匹配。

### 处理顺序

提示词处理的完整顺序：

1. **EJS 渲染**（st-prompt-template）
2. **占位符替换**（$变量）
3. **随机数生成**（`<random>` 标签）
4. **条件模板解析**（`<if>` 标签）

### 示例：完整的随机事件系统

```
<prologue>
# 📜 第X幕：[场景名]

**【随机事件判定】**
🎲 **本轮随机数**：<random min="1" max="100" />

<if cell="状态表/系统/当前模式 == 日常">
**基于随机数的事件建议**：
- 若随机数 1-10：建议触发重大意外（如敌人突袭、自然灾害）
- 若随机数 11-30：建议触发小波折（如物品损坏、NPC拒绝）
- 若随机数 31-70：平稳推进，无特殊事件
- 若随机数 71-90：建议触发小机遇（如发现隐藏物品、获得好感）
- 若随机数 91-100：建议触发重大机遇（如意外收获、关键线索）
</if>
</prologue>
```
