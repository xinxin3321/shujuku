# 数据库条件模板语法说明

## 概述

数据库插件提供了条件模板功能，允许根据条件动态显示或隐藏提示词内容。该功能通过 `<if>` 标签实现，支持关键词匹配、表格数值比较以及统一条件表达式三种条件类型。此外，还提供随机数生成功能。

## 功能开关

在"数据库更新"设置区域，勾选"启用条件模板功能"复选框即可开启此功能。

## 基本语法

### 1. 关键词匹配 (`seed`)

根据数据库表格内容（除纪要表外）和 `$6`（上轮剧情规划数据）中是否包含指定关键词来决定是否显示内容。

```
<if seed="关键词表达式">条件内容</if>
```

### 2. 表格数值比较 (`cell`)

根据数据库表格中的数值来决定是否显示内容。

```
<if cell="表格名/行名/列名 比较运算符 数值">条件内容</if>
```

### 3. 统一条件表达式 (`cond`) 【新增】

支持多条件组合和括号分组，可以混合使用 `seed:` 和 `cell:` 条件。

```
<if cond="条件表达式">条件内容</if>
```

### 4. 随机数生成 (`random`)

在提示词中生成指定范围内的随机整数。

```
<random min="最小值" max="最大值" />
```

**示例**：
```
<!-- 生成 1-100 之间的随机数 -->
🎲 本轮随机数：<random min="1" max="100" />

<!-- 生成 1-6 之间的随机数（模拟骰子） -->
🎲 骰子结果：<random min="1" max="6" />
```

## 关键词匹配详解

### 基本用法

```
<if seed="战斗">这里的内容只有在消息中包含"战斗"时才会显示</if>
```

### 逻辑运算符

| 运算符 | 含义 | 示例 |
|--------|------|------|
| `&` | 与（AND） | `战斗&魔法` - 同时包含两个关键词 |
| `,` | 或（OR） | `战斗,魔法` - 包含任意一个关键词 |
| `!` | 非（NOT） | `!战斗` - 不包含该关键词 |

### 组合示例

```
<!-- 同时包含"战斗"和"魔法" -->
<if seed="战斗&魔法">
  你正在施展战斗魔法！
</if>

<!-- 包含"战斗"或"逃跑" -->
<if seed="战斗,逃跑">
  场景变得紧张起来。
</if>

<!-- 不包含"安全" -->
<if seed="!安全">
  危险潜伏在四周。
</if>

<!-- 复杂组合：包含("战斗"且"魔法") 或 ("逃跑"且"追击") -->
<if seed="战斗&魔法,逃跑&追击">
  这是一场追逐战！
</if>
```

### 模糊匹配

关键词匹配支持模糊匹配，会自动匹配包含该词的更长词语：

- 关键词 `战斗` 可以匹配 `战斗中`、`战斗开始`、`激烈战斗` 等
- 匹配不区分大小写

## 表格数值比较详解

### 基本语法

```
<if cell="表格名/行名/列名 比较运算符 数值">条件内容</if>
```

### 比较运算符

| 运算符 | 含义 | 示例 |
|--------|------|------|
| `>` | 大于 | `> 50` |
| `<` | 小于 | `< 100` |
| `>=` | 大于等于 | `>= 30` |
| `<=` | 小于等于 | `<= 70` |
| `==` | 等于 | `== 100` |
| `!=` | 不等于 | `!= 0` |

### 示例

```
<!-- 魔力值大于30时显示 -->
<if cell="状态表/主角/魔力值 > 30">
  你有足够的魔力施放高级魔法。
</if>

<!-- 好感度小于等于30时显示 -->
<if cell="关系表/陈默/好感度 <= 30">
  陈默对你保持警惕。
</if>

<!-- 等级等于10时显示 -->
<if cell="属性表/主角/等级 == 10">
  恭喜你达到了10级！
</if>

<!-- 字符串相等比较（支持中文） -->
<if cell="全局数据表/本轮故事模式 == 战斗模式">
  当前处于战斗状态！
</if>

<!-- 字符串不等比较 -->
<if cell="全局数据表/本轮故事模式 != 战斗模式">
  当前不是战斗状态。
</if>
```

### 行列颠倒自动匹配

如果指定的"行名/列名"找不到，系统会自动尝试颠倒查找：

```
<!-- 以下两种写法等效 -->
<if cell="状态表/主角/魔力值 > 30">...</if>
<if cell="状态表/魔力值/主角 > 30">...</if>
```

### 单行表格支持

对于只有一行数据的表格（如「全局数据表」），系统会自动识别并支持直接用列名获取值：

```
<!-- 全局数据表只有一行数据，可以直接用「表格名/列名」格式 -->
<if cell="全局数据表/本轮故事模式 == 战斗模式">
  当前处于战斗状态！
</if>

<!-- 也可以用「表格名/任意标识/列名」格式，系统会自动找到对应的列 -->
<if cell="全局数据表/状态/本轮故事模式 == 战斗模式">
  当前处于战斗状态！
</if>
```

### 字符串比较说明

- 字符串比较支持 `==`（相等）和 `!=`（不等）运算符
- 字符串比较是**精确匹配**，区分大小写
- 如果单元格值是数字类型，比较值也是数字，则进行数值比较
- 否则进行字符串比较

## 统一条件表达式详解 (`cond`) 【新增】

### 基本语法

统一条件表达式支持在单个 `<if>` 标签中组合多个条件，使用 `seed:` 和 `cell:` 前缀指定条件类型。

```
<if cond="条件表达式">
  条件满足时显示的内容
<else>
  条件不满足时显示的内容
</if>
```

### 子条件类型

| 前缀 | 说明 | 示例 |
|------|------|------|
| `seed:` | 关键词匹配 | `seed:战斗` |
| `cell:` | 表格数值比较 | `cell:状态表/主角/魔力值 > 30` |

### 逻辑运算符

| 运算符 | 含义 | 示例 |
|--------|------|------|
| `&` | 与（AND） | `seed:战斗 & cell:魔力 > 30` |
| `,` | 或（OR） | `seed:战斗 , seed:休息` |
| `!` | 取反（NOT） | `!seed:安全` |
| `()` | 括号分组 | `(seed:战斗 & cell:魔力 > 30) , cell:好感度 > 80` |

**注意**：`!` 可以对单个条件或括号表达式整体取反：
- `!seed:安全` - 不包含"安全"
- `!(seed:战斗 & cell:魔力 > 30)` - 不是(战斗且魔力充足)

### 运算优先级

1. **括号 `()`** - 最高优先级，先计算括号内
2. **`!` (NOT)** - 次高优先级
3. **`&` (AND)** - 再次
4. **`,` (OR)** - 最低优先级

### 示例

#### 多个表格条件组合

```
<!-- 两个表格值共同约束（AND） -->
<if cond="cell:关系表/陈默/好感度 > 50 & cell:资产表/陈默/财富值 > 1000">
  陈默对你有好感且富有。
</if>

<!-- 三个表格条件组合 -->
<if cond="cell:关系表/陈默/好感度 > 50 & cell:资产表/陈默/财富值 > 1000 & cell:状态表/陈默/健康 > 60">
  陈默对你有好感、富有且健康。
</if>

<!-- OR 组合 -->
<if cond="cell:关系表/陈默/好感度 > 80 , cell:资产表/陈默/财富值 > 5000">
  陈默对你很有好感，或者非常富有。
</if>
```

#### seed 和 cell 混合条件

```
<!-- 战斗场景且魔力充足 -->
<if cond="seed:战斗 & cell:状态表/主角/魔力值 > 30">
  战斗中且魔力充足，可以施放高级魔法。
</if>

<!-- 包含"商店"或"交易" 且 金币>100 -->
<if cond="seed:商店,交易 & cell:资产表/主角/金币 > 100">
  你有足够的金币购物。
</if>

<!-- 不包含"安全" 且 生命值<=30 -->
<if cond="!seed:安全 & cell:状态表/主角/生命值 <= 30">
  危险！你的生命值很低！
</if>
```

#### 括号分组控制优先级

```
<!-- (战斗且魔力充足) 或 好感度很高 -->
<if cond="(seed:战斗 & cell:状态表/主角/魔力值 > 30) , cell:关系表/陈默/好感度 > 80">
  战斗中魔力充足，或者陈默对你很有好感。
</if>

<!-- 战斗 且 (魔力充足 或 好感度很高) -->
<if cond="seed:战斗 & (cell:状态表/主角/魔力值 > 30 , cell:关系表/陈默/好感度 > 80)">
  战斗中，要么魔力充足，要么陈默对你很有好感。
</if>

<!-- 复杂嵌套 -->
<if cond="(seed:战斗 & cell:魔力 > 30) , (seed:休息 & cell:生命 < 50)">
  (战斗且魔力充足) 或 (休息且生命值低)
</if>
```

#### 取反条件

```
<!-- 对单个条件取反 -->
<if cond="!seed:安全 & cell:生命值 <= 30">
  危险！不在安全区域且生命值很低！
</if>

<!-- 对括号表达式整体取反 -->
<if cond="!(cell:关系表/陈默/好感度 > 50 & cell:资产表/陈默/财富值 > 1000)">
  陈默对你没有好感或穷。
</if>

<!-- 取反的逻辑解释 -->
<!-- !(A & B) = !A , !B (德摩根定律) -->
<!-- 上面的条件等价于：好感度≤50 或 财富≤1000 -->

<!-- 对 OR 表达式取反 -->
<if cond="!(seed:战斗 , seed:休息)">
  既不在战斗中，也不在休息。
</if>
```

### else 的含义

`else` 表示**整个条件表达式不满足**时显示的内容：

| 条件表达式 | 满足时显示 | else 显示 |
|-----------|----------|----------|
| `seed:战斗 & cell:魔力 > 30` | 战斗中且魔力>30 | 不在战斗中 **或** 魔力≤30 |
| `seed:战斗 , cell:魔力 > 30` | 战斗中 **或** 魔力>30 | 不在战斗中 **且** 魔力≤30 |

### 与原有语法的对比

**之前（繁琐的嵌套写法）**：
```
<if seed="战斗">
  <if cell="状态表/主角/魔力值 > 30">
    <if cell="关系表/陈默/好感度 > 50">
      战斗中，魔力充足，陈默愿意帮助你。
    <else>
      战斗中，魔力充足，但陈默对你冷淡。
    </if>
  <else>
    战斗中，魔力不足。
  </if>
</if>
```

**之后（简洁的单行写法）**：
```
<if cond="seed:战斗 & cell:状态表/主角/魔力值 > 30 & cell:关系表/陈默/好感度 > 50">
  战斗中，魔力充足，陈默愿意帮助你。
</if>
<if cond="seed:战斗 & cell:状态表/主角/魔力值 > 30 & cell:关系表/陈默/好感度 <= 50">
  战斗中，魔力充足，但陈默对你冷淡。
</if>
<if cond="seed:战斗 & cell:状态表/主角/魔力值 <= 30">
  战斗中，魔力不足。
</if>
```

## else 分支

当条件不满足时，可以显示备选内容：

```
<if seed="战斗">
  战斗场景内容
<else>
  非战斗场景内容
</if>
```

```
<if cell="状态表/主角/魔力值 > 30">
  魔力充足，可以施放高级魔法。
<else>
  魔力不足，只能使用普通攻击。
</if>
```

## 嵌套条件

条件可以嵌套使用，实现更复杂的逻辑：

```
<if seed="战斗">
  <if cell="状态表/主角/魔力值 > 30">
    你施放了一个强大的魔法攻击！
  <else>
    魔力不足，你只能使用普通攻击。
  </if>
<else>
  <if seed="休息">
    你找了一个安静的地方休息。
  <else>
    你继续探索这个区域。
  </if>
</if>
```

**注意**：嵌套深度有限制，建议不超过 5 层。

## 完整示例

### 示例1：根据场景类型显示不同内容

```
# 场景描述

<if seed="战斗">
  ## 战斗场景
  当前处于战斗状态，请描述你的战斗行动。
  <if cell="状态表/主角/生命值 <= 30">
    ⚠️ 警告：你的生命值较低！
  </if>
<else>
  <if seed="商店">
    ## 商店场景
    你可以购买或出售物品。
  <else>
    ## 探索场景
    你正在探索这个区域。
  </if>
</if>
```

### 示例2：根据角色状态显示人设

```
# 角色状态

<if cell="属性表/陈默/堕落值 <= 10">
  ## 阶段1：迷茫的旁观者
  陈默保持着纯真的心态，对世界充满好奇。
<else>
  <if cell="属性表/陈默/堕落值 <= 30">
    ## 阶段2：初涉黑暗
    陈默开始感受到内心的变化。
  <else>
    <if cell="属性表/陈默/堕落值 <= 60">
      ## 阶段3：黑暗觉醒
      陈默的内心逐渐被黑暗侵蚀。
    <else>
      ## 阶段4：堕落完成
      陈默已经完全堕入黑暗。
    </if>
  </if>
</if>
```

### 示例3：复杂条件组合

```
# 剧情分支

<if seed="战斗&魔法">
  <if cell="状态表/主角/魔力值 > 50">
    你施放了一个强大的范围魔法，敌人受到重创！
  <else>
    你尝试施放魔法，但魔力不足，效果大打折扣。
  </if>
<else>
  <if seed="战斗,逃跑">
    <if cell="状态表/主角/速度 > 敌人/速度">
      你成功逃脱了战斗！
    <else>
      敌人追上了你，战斗无法避免。
    </if>
  <else>
    平静的时光流逝...
  </if>
</if>
```

## 与 st-prompt-template 的关系

数据库插件的条件模板功能与酒馆的 st-prompt-template 插件协同工作：

1. **处理顺序**：提示词先经过 st-prompt-template 处理（EJS 语法），再由数据库的条件模板处理（`<if>` 语法）
2. **功能互补**：
   - st-prompt-template：处理 `<% %>` EJS 语法，可访问酒馆变量系统
   - 数据库条件模板：处理 `<if>` 语法，可访问数据库表格数据

### 示例：两者配合使用

```
<%# 先由 st-prompt-template 处理 EJS 语法 %>
<% if (getvar("some_var") === "value") { %>
  这里的内容会先被 EJS 处理
<% } %>

<%# 再由数据库条件模板处理 %>
<if seed="战斗">
  这里的内容会被条件模板处理
</if>
```

## 注意事项

1. **标签闭合**：所有 `<if>` 标签必须正确闭合 `</if>`
2. **大小写敏感**：关键词匹配不区分大小写，表格名/行名/列名区分大小写
3. **数值类型**：表格数值比较时，非数值内容会被视为 0
4. **性能考虑**：复杂的嵌套条件可能影响处理速度，建议控制嵌套深度
5. **错误处理**：语法错误时，整个 `<if>` 块会被保留原样输出

## 调试技巧

1. 检查控制台日志，查看条件匹配结果
2. 使用简单的测试条件验证功能是否正常
3. 逐步添加复杂条件，确保每一步都正确工作

## 随机数生成详解

### 基本语法

```
<random min="最小值" max="最大值" />
```

### 功能说明

- 在提示词处理时生成一个指定范围内的随机整数
- 生成的随机数会替换整个 `<random>` 标签
- 随机数在每次剧情推进时重新生成
- **新增**：支持 `id` 属性存储随机数变量，可在条件中引用

### 随机数变量（id 属性）【新增】

使用 `id` 属性可以将随机数存储为变量，在后续文本或条件中引用：

```
<!-- 生成随机数并存储为变量 -->
<random id="event" min="1" max="100" />
<random id="dice" min="1" max="6" />

<!-- 在文本中引用随机数变量 -->
🎲 **事件随机数**：$random:event
⚔️ **攻击骰子**：$random:dice
```

**注意**：带 `id` 的 `<random>` 标签不会在原位置显示随机数，需要用 `$random:id` 引用。

### 随机数条件判断（random: 前缀）【新增】

在统一条件表达式中，可以使用 `random:` 前缀进行随机数条件判断。

#### 格式1：引用随机数变量

```
<!-- 引用已生成的随机数变量 -->
<random id="dice" min="1" max="6" />

<if cond="random:dice > 4">
  🎯 骰子点数大于4，攻击成功！
<else>
  ❌ 骰子点数不大于4，攻击失败。
</if>
```

#### 格式2：内联随机数

直接在条件中生成随机数并判断，无需预先定义：

```
<!-- 内联随机数：生成1-100的随机数并判断 -->
<if cond="random:1-100 <= 10">
  ⚠️ 触发重大意外事件！
<else>
  <if cond="random:1-100 >= 90">
    🌟 触发重大机遇事件！
  <else>
    ✨ 平稳推进。
  </if>
</if>
```

### 使用场景

#### 1. 随机事件判定

```
## 随机事件判定
<random id="event" min="1" max="100" />
🎲 **本轮随机数**：$random:event

<if cond="random:event <= 10">
  ⚠️ 触发重大意外事件（如敌人突袭、自然灾害）
<else>
  <if cond="random:event <= 30">
    📊 触发小波折（如物品损坏、NPC拒绝）
  <else>
    <if cond="random:event >= 90">
      🌟 触发重大机遇（如意外收获、关键线索）
    <else>
      <if cond="random:event >= 70">
        🍀 触发小机遇（如发现隐藏物品、获得好感）
      <else>
        ✨ 平稳推进，无特殊事件
      </if>
    </if>
  </if>
</if>
```

#### 2. 骰子模拟

```
<!-- D20 骰子 -->
🎲 D20: <random min="1" max="20" />

<!-- D6 骰子（3个） -->
🎲 3D6: <random min="1" max="6" /> + <random min="1" max="6" /> + <random min="1" max="6" />

<!-- 带判定的骰子 -->
<random id="attack" min="1" max="20" />
⚔️ 攻击检定：$random:attack

<if cond="random:attack >= 15">
  🎯 命中！造成全额伤害。
<else>
  <if cond="random:attack >= 10">
    📊 擦边命中，造成半数伤害。
  <else>
    ❌ 未命中。
  </if>
</if>
```

#### 3. 随机选择

```
<!-- 从 1-4 中随机选择一个选项 -->
随机选项编号：<random min="1" max="4" />

<!-- 带条件的随机选择 -->
<random id="choice" min="1" max="4" />
随机选项：$random:choice

<if cond="random:choice == 1">
  选项A：前往森林
<else>
  <if cond="random:choice == 2">
    选项B：前往城镇
  <else>
    <if cond="random:choice == 3">
      选项C：留在原地
    <else>
      选项D：返回营地
    </if>
  </if>
</if>
```

#### 4. 与其他条件组合

```
<!-- 战斗判定：场景关键词 + 随机骰子 -->
<random id="battle_dice" min="1" max="20" />

<if cond="seed:战斗 & random:battle_dice >= 15">
  🎯 战斗中，骰子点数高，攻击成功！
<else>
  <if cond="seed:战斗 & random:battle_dice >= 10">
    📊 战斗中，骰子点数中等，造成部分伤害。
  <else>
    <if cond="seed:战斗">
      ❌ 战斗中，骰子点数低，攻击失败。
    <else>
      ⚔️ 不在战斗中。
    </if>
  </if>
</if>
```

### 处理顺序

提示词处理的完整顺序：

1. **EJS 渲染**（st-prompt-template）
2. **占位符替换**（$变量）
3. **随机数生成**（`<random>` 标签，存储变量）
4. **随机数变量替换**（`$random:id` 引用）
5. **条件模板解析**（`<if>` 标签，支持 `random:` 条件）
