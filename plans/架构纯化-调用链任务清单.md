# 架构纯化 — 调用链任务清单

## 原则

**每一层只做自己该做的事，不越权。**

### service 层

函数体内所有操作只允许属于以下四类：
1. 读数据（调用 data 层或 shared 层获取数据）
2. 业务决策（基于数据的 if/else/switch 逻辑）
3. 写数据（调用 data 层持久化结果）
4. 构造返回值

函数的所有产出都通过返回值传递给调用方。函数体内出现不属于这四类的操作 = 越权。

函数返回 void = 可疑——它的产出去哪了？要么是通过副作用偷偷做了调用方该做的事，要么是吞掉了调用方需要的信息。

### data 层

函数体内所有操作只允许属于以下三类：
1. 接收参数（key、value、查询条件）
2. 读/写存储（localStorage、IndexedDB、聊天记录字段、酒馆 API）
3. 返回数据

函数体内出现不属于这三类的操作 = 越权。具体来说：
- 基于业务规则的 if/else 分支 = 越权（业务决策是 service 层的事）
- 编排多步流程（读→判断→写→刷新→通知）= 越权
- 通过 _injectXxxDeps 接收 service/presentation 层的依赖 = 越权
- 持有可变全局业务状态（导出 let 被多个文件直接读写）= 越权

### shared 层

只允许包含：
1. 纯函数（输入→输出，无副作用、无 IO、不依赖运行时状态）
2. 常量
3. 类型定义

import 只允许来自 shared 层内部或外部库。import service/data/presentation 层 = 违规。

### presentation 层

1. 调用 service 层函数
2. 拿到结构化结果
3. 根据结果自己决定 UI 行为（弹 toast、刷新列表、操作 DOM、绑定事件、触发宿主 UI 事件、构造 HTML）

presentation 层是唯一允许做 UI 操作的层。

### 违规判定

函数体内出现不属于该层允许操作的行为 = 违规。

不管是直接调用、改名调用、下沉到其他层调用、还是通过回调/依赖注入传入后调用——只要行为不属于该层的允许操作，就是违规。形式不重要，行为才重要。

## 验收方式

每条任务改完后，在验收报告中用同一编号，对链路上**每个改动函数**做同一件事：

**列出函数体内的每一个操作，标明它属于该层允许的哪一类。标不上的 = 违规 = 没改干净。**

格式：

```
T-XX: 链路名称

  【service 层】functionName (文件名.ts)
    改后签名: functionName(params): 返回值类型
    逐项打勾:
      ☐ ①读数据 — 函数体内的读数据操作: L行号 xxx_ACU() / 无
      ☐ ②业务决策 — 函数体内的决策操作: L行号 if(xxx) / 无
      ☐ ③写数据 — 函数体内的写数据操作: L行号 yyy_ACU(data) / 无
      ☐ ④构造返回值 — 函数体内的 return: L行号 return { ... } / 无
      ☐ ⑤除①②③④外还有没有其他操作？— 无 ✅ / 有: L行号 xxx ❌

  【data 层】functionName (文件名.ts)
    改后签名: functionName(params): 返回值类型
    逐项打勾:
      ☐ ①接收参数 — L行号 / 无
      ☐ ②读/写存储 — L行号 store.setItem() / 无
      ☐ ③返回数据 — L行号 return data / 无
      ☐ ④除①②③外还有没有其他操作？— 无 ✅ / 有: L行号 xxx ❌

  【shared 层】functionName (文件名.ts)
    逐项打勾:
      ☐ ①纯计算(输入→输出) — L行号 / 无
      ☐ ②import 来源 — 逐行列出，全部来自 shared/外部库 ✅ / 有来自 service/data ❌
      ☐ ③除①外还有没有 IO、副作用、运行时状态依赖？— 无 ✅ / 有: L行号 xxx ❌

  【presentation 层】调用方 (文件名.ts)
    逐项打勾:
      ☐ ①调用 service 层 — L行号 const result = serviceFunc()
      ☐ ②拿结果做 UI 操作 — L行号 showToastr(result.message) / 刷新列表 / ...
      ☐ ③除①②外还有没有直接读写 data 层、做业务决策等越权操作？— 无 ✅ / 有: L行号 xxx ❌
    改前: serviceFunc();
    改后: const result = serviceFunc(); showToastr(result.message);
```

**每个 ☐ 必须逐个打 ✅ 或 ❌。最后一项"除允许类别外还有没有其他操作"必须明确回答。**

有 ❌ = 没改干净。全 ✅ = 通过。

任务清单有 T-01~T-23，验收报告也必须有 T-01~T-23。缺了 = 跳过了。
